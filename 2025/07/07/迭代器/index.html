惰性初始化

这种惰性初始化的方式确保了创建迭代器不会有任何额外的性能损耗，其中的元素也不会被消耗，只有使用到该迭代器的时候，一切才开始。

next方法

next() 是 Iterator trait 中定义的核心方法，用来逐个返回序列中的下一个元素，直到没有为止。

返回值为什么是option

因为你不确定迭代器还有没有元素了。

IntoIterator 特征

IntoIterator 是一个 trait，表示一个类型可以转换成一个迭代器（Iterator）。

Rust 有三种方式可以获得迭代器

  • v.iter()：返回 &T 类型的迭代器
  • v.iter_mut()：返回 &mut T 类型的迭代器
  • v.into_iter()：获取所有权，消费掉集合，产生一个“值”的迭代器

Iterator 和 IntoIterator 的区别

Iterator 就是迭代器特征，只有实现了它才能称为迭代器，才能调用 next。而 IntoIterator 强调的是某一个类型如果实现了该特征，它可以通过 into_iter，iter 等方法变成一个迭代器。

适配器

消费者适配器

消费者适配器（consuming adaptors） 是指会消费掉迭代器本身的方法，即它们会拿走 self 的所有权，最终返回一个非迭代器类型的结果（比如 i32, String, Vec<T> 等）。

常见的消费者适配器（重点）
方法 返回类型 说明
.collect() Vec<T>, HashSet<T> 等 把迭代器“收集”成集合
.sum() i32, f64, String 等 所有元素相加（实现了 Sum）
.product() 数值类型 所有元素相乘（实现了 Product）
.count() usize 统计元素个数
.last() Option<T> 返回最后一个元素
.nth(n) Option<T> 返回第 n 个元素
.max() / .min() Option<T> 返回最大/最小值（需 Ord）
.any() / .all() bool 是否存在 / 是否全部符合条件
.find() Option<T> 找到第一个满足条件的元素
.fold(init, f) 任意类型 自定义折叠累加器
.for_each() () 对每个元素执行操作（通常用于副作用）

  • 一旦你调用了消费者适配器，比如 .sum()、.collect()，迭代器就不能再用了；
  • 所以 .map().sum() ✅，但不能 .sum().map() ❌（因为 .sum() 已经返回值了，不是迭代器了）；
  • 消费者适配器返回的是一个具体结果，不再是惰性求值。

迭代器适配器

它们不会消费掉原始迭代器，而是返回一个新的、可以继续迭代的东西。属于惰性计算（lazy evaluation），只有配合消费者适配器时才会真正执行。

常见的迭代器适配器方法
方法 作用说明 示例
.map(f) 对每个元素应用函数 f `.map( x x * 2)`
.filter(p) 只保留满足条件的元素 `.filter( x x % 2 == 0)`
.enumerate() 每个元素带上索引 (index, value) .enumerate()
.take(n) 只取前 n 个元素 .take(3)
.skip(n) 跳过前 n 个元素 .skip(2)
.chain(it2) 将两个迭代器串联起来 .chain(other_iter)
.zip(it2) 将两个迭代器“打包”成元组 .zip(1..)
.rev() 倒序（要求实现 DoubleEndedIterator） .rev()
.flat_map() 映射后展开嵌套迭代器 `.flat_map( x x..x+2)`
.inspect() 调试用：对每个元素执行副作用函数 `.inspect( x println!(“{}”, x))`

它们只是在构建一个操作链条（pipeline），不立即执行。只有最后有一个“消费者适配器”比如 .collect(), .sum() 等，才会触发实际迭代。
