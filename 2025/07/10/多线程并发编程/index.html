这表示：主线程会阻塞等待子线程执行完成，然后继续执行。

有多个子线程

  • Rust 的线程是“非阻塞启动的”；
  • 如果主线程退出了，程序就终止了，子线程可能还没来得及完成就被强制杀死；
  • 所以你必须 join() 才能确保子线程逻辑执行完整。

在线程闭包中使用 move

在 Rust 中，当你在 线程闭包中使用外部变量（比如 thread::spawn(...) 内部），**必须加 ****move**

为什么要加 move

🔒 **move**** 把变量的所有权移动进闭包**，让线程拥有它，防止生命周期/所有权问题。

Rust 的线程是异步运行的，所以不能默认借用外部变量，否则主线程可能先退出、数据释放了，子线程还在用，会导致崩溃或编译错误。

正确写法：加 move

为什么是 move 而不是借用

因为：

  1. 线程执行是异步的，主线程不能保证变量仍然活着；
  2. Rust 编译器不允许跨线程借用局部变量；
  3. move 是 Rust 中让闭包拥有变量的方式，闭包会自动捕获它用到的值，并把它变成自己所有

总结

场景 是否需要 move
闭包中使用外部变量 ✅ 必须使用 move
不使用外部变量，只打印常量等 ❌ 不需要

线程崩溃（panic）

如果线程中的代码发生 panic!()，该线程会立即终止，但不会影响主线程或其他线程，除非你主动处理：

主线程结束时，子线程的影响

  • 如果主线程先退出，程序整体会终止，未 join 的子线程可能被强制中止（线程未完成）。
  • 所以推荐：子线程要 handle.join()，确保完成。
判断线程是否完成的唯一方式：join()
