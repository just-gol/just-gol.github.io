线程间的消息传递

Created2025-07-10|Updated2025-07-10
|Post Views:

在 Rust 中，线程间的消息传递（message passing）是一种推荐的并发方式，遵循的是：

✅ “不要共享内存，而是共享消息” 的并发模型（类似 Go 的 channel）。

Rust 提供了非常强大且安全的标准库模块：

🔧 std::sync::mpsc

用于多线程安全的消息传递

基本概念

  • mpsc：multi-producer, single-consumer（多个发送者，一个接收者）
  • 使用 channel() 创建一对：
    • 发送端 Sender
    • 接收端 Receiver

在 std::sync::mpsc::Receiver 上的方法：

尝试接收一条消息，如果没有消息，**立即返回 ****Err(TryRecvError::Empty)**，不会阻塞当前线程。

总结

方法 说明
send(val) 往通道发送值，可阻塞或立即返回
recv() 阻塞直到收到消息
try_recv() 不阻塞，如果没消息直接返回 Err
iter() 将通道作为迭代器持续接收消息

传输具有所有权的数据

Rust 的 mpsc::channel 采用 send(T) 方式传递数据，会转移所有权 到接收方线程。

这和普通函数参数传递一致：不是共享、不是引用，而是值移动（move）。

示例

传输一个有所有权的结构体

  • send(book)：把数据发送到 channel 的同时，把 book 的所有权移动过去了；
  • recv()：接收方接收数据后，获得其所有权；
  • 所以：接收方是这个值的新主人。
如果想发送多个值：用 clone 或 for 循环 + move
可以传什么类型

只要满足 T: Send + 'static 的类型都可以发送（包括结构体、Vec、String、Box、Arc 等）。

不能发送的类型
  • Rc<T>（不是线程安全的）
  • 引用类型（如 &str）：因为引用不能移动，生命周期难以保证

✅ 替代方式：

  • 用 String 替代 &str
  • 用 Arc<T> 替代 Rc<T>

异步通道

异步通道 = 支持 async/await 的 channel，用于 异步任务之间的通信，不阻塞线程。

适用于：

  • 使用 tokio、async-std 等异步运行时
  • 多个异步任务间发送消息
  • 高并发但无需线程阻塞的场景

同步通道

示例

同步通道，无缓冲（阻塞发送）

  • 发送线程被阻塞直到主线程调用 recv()
  • 确保发送和接收步调一致

同步通道，有缓冲（类似异步通道）

对比：channel() vs sync_channel()
通道类型 函数名 是否阻塞发送 是否支持缓冲 应用场景
异步通道 mpsc::channel() ❌ 否（非阻塞） ❌ 没缓冲 多线程发送日志/数据，灵活处理
同步通道 mpsc::sync_channel(n) ✅ 是（受缓冲限制） ✅ 支持缓冲 多线程强同步，生产消费模型

缓冲区的通道

缓冲区是一个 临时存储空间，用于存放“已经发送，但还没有被接收的数据”。

  • 类似一个中间仓库，生产者把数据放进去，消费者从中取出。
  • 它的容量（大小）决定了通道是否会立刻阻塞发送方。
示例

想象你是一位厨师（发送者），顾客（接收者）点菜。

  • 你把做好的菜放到窗口（缓冲区）
  • 窗口最多只能放 3 个菜（缓冲区大小为 3）

情况 1：窗口没满

  • 顾客还没来拿菜，你可以继续放 → ✅ 不阻塞

情况 2：窗口满了

  • 顾客还没来，你就只能等他拿走再放 → ❌ 发送阻塞
Rust 中的缓冲区

在 Rust 的 mpsc::sync_channel(buffer_size) 中：

  • buffer_size = 0：无缓冲通道，发送立即阻塞，直到接收者收到
  • buffer_size = n (>0)：有缓冲通道，最多能存放 n 个消息
    • 只当缓冲区满时，send() 才会阻塞
    • recv() 会从缓冲区取消息，腾出空间
示例：
总结表格
缓冲区大小 表示 send()行为
0 无缓冲通道 立刻阻塞，直到接收方接收
> 0 有缓冲通道 直到缓冲区满才会阻塞
很大 类似异步行为 多次 send 都不会阻塞
用途场景
缓冲大小 应用场景
0 精准同步、一发一收（线程步调配合）
1~n 控制节奏、避免发送方“超前”发送太多
大缓冲 快速发消息，不关心节奏
