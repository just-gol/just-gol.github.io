闭包

Created2025-07-05|Updated2025-07-05
|Post Views:

闭包是可以捕获其 环境中变量 的匿名函数。它的语法类似函数，但更简洁，也更灵活。

基本语法

最简单的闭包

三种闭包

Fn

只读借用环境变量，不改变它。

举例

类型 T 必须是一个可以 被调用的类型（函数或闭包），其接受一个 u32 类型的参数，并返回一个 u32 类型的值。

部分 含义
T 泛型类型参数，可能是一个闭包或函数
Fn(u32) -> u32 表示 T 必须实现 Fn 这个 trait，也就是：可调用、接收一个 u32 参数、返回一个 u32 结果

(self.query)(args) 而不是 self.query(args)?

  • self.query 是结构体的字段（不是一个方法），
  • 如果你直接写 self.query(args)，Rust 会误以为你想调用一个名为 **query** 的方法。
  • query 是结构体字段，类型是 T，而 T 必须满足 Fn(u32) -> u32。
  • 所以 query 是一个可以接收 **u32** 并返回 **u32** 的闭包或函数。

FnOnce

该类型的闭包会拿走被捕获变量的所有权。Once 顾名思义，说明该闭包只能运行一次

解释:

  • F 是一个闭包类型；
  • 它可以被调用（FnOnce），接受一个 usize 参数，返回 bool；
  • 并且这个闭包本身实现了 **Copy** trait，也就是说，你可以复制这个闭包变量本身。
  • func 是一个闭包变量；
  • 调用了 两次；
  • 如果 F: FnOnce，默认只能调用一次，因为闭包可能 move 进来的外部变量；
  • 但是你加了 Copy 后，Rust 允许编译器复制 **func**，然后各调用一遍副本，不再 move 原始闭包！

FnMut

FnMut 是一种可以多次调用，它以可变借用的方式捕获了环境中的值

  • 闭包里使用了外部变量 counter；
  • 它对 counter 执行了 += 1，这是“可变借用”；
  • 所以闭包类型被自动推断为 FnMut；
  • 函数 call_fn_mut 必须要求 F: FnMut() 才能接受它。

三种 Fn 的关系

闭包最少是 FnOnce，不修改变量就是 Fn，修改变量就是 FnMut。

  • 每个闭包 **至少是 ****FnOnce**；
  • 如果闭包不 move 捕获的变量，**还能自动实现 ****FnMut**；
  • 如果闭包还不修改捕获的变量，**还能自动实现 ****Fn**。

闭包只是对 s 进行了不可变借用，实际上，它可以适用于任何一种 Fn 特征：三个 exec 函数说明了一切。强烈建议读者亲自动手试试各种情况下使用的 Fn 特征

总结

Trait 适用于闭包… 捕获外部变量方式 能调用几次
Fn 不修改外部变量 不可变借用 &T 多次 ✅
FnMut 修改外部变量 可变借用 &mut T 多次 ✅
FnOnce move 外部变量 拿走所有权 T 只能一次 ❌

疑问的代码

疑问代码一

  • Rust 默认将闭包变量 update_string 推断为 Fn；
  • Fn 要求只捕获不可变引用（&s），
  • 但你闭包里调用了 s.push_str(...)，这是可变操作，需要 &mut s；
  • 所以 冲突了：Rust 不允许一个默认不可变借用的闭包里去执行可变借用操作

疑问代码二

Rust 规定：调用 FnMut 类型的闭包，必须使用 **mut** 变量，但这个 mut 可以是：

  • 主函数中写 let mut closure
  • 或者传进函数后，在参数上写 mut f

只要闭包最终是在一个 **mut** 变量上被调用的，就合法。

疑问代码三

因为 **exec** 要求的是不可变闭包 **Fn**（不是 **FnMut** 或 **FnOnce**），所以 Rust 推断闭包只能对 **s** 做不可变借用（**&s**），从而不会发生所有权转移

如果闭包不是只读，就不能这么干了

Rust 会自动根据你对变量的使用情况，推断闭包使用最弱的 trait（Fn ➝ FnMut ➝ FnOnce）。只读变量 ➝ &s ➝ 不会发生所有权转移。

疑问代码四

加了 move，为什么 s 被 move 进闭包了，但 exec(f) 还能用？函数签名 F: Fn() 不应该不能接受 move 的闭包吗？为啥没报错？

  • s 被 move 进闭包；
  • 但你对 s 的使用是 println!("{}", s)，只是只读（不可变借用）；
  • 所以即使 s 已经不在 main 中，闭包内部还是可以多次读取它；
  • 这个闭包满足 Fn 的要求：只读访问闭包内字段，能多次调用；
  • 所以可以传进 exec<F: Fn()>。

Rust 会推断闭包实现哪个 trait：

  • 只读变量 → Fn
  • 修改变量 → FnMut
  • 使用变量（比如返回 s 或 move 掉）→ FnOnce

你的闭包只打印 s，没有消耗掉 s，所以仍然是 Fn。

总结

场景 要不要 mut
闭包被你直接调用（比如 update_string("hi")） 你要写 let mut update_string
闭包传入函数，函数内部调用（写了 mut f） 不需要你写 mut，函数负责
闭包是 Fn 类型（不修改捕获变量） 不需要 mut
闭包是 FnOnce（可能 move） 一次调用即可，无需 mut，但不能再用
