包（Package） 和 库（Crate）

Package（包）

就是一个完整的项目文件夹，里面包含了代码、依赖、配置。

一个包可以：

  • 是一个可运行的程序（比如 main.rs）
  • 或者是一个库（比如 lib.rs）
  • 甚至两个都有

Crate（库）

Crate 是 Rust 编译时的最小单元，也就是最终被编译出来的一个“产物”

也可以理解为：

  • 如果说 Package 是房子
  • 那 Crate 就是房子里的某个“房间”，可以单独使用

一个包可以包含 一个或多个 Crate，常见的有两类：

Crate 类型 文件入口 说明
Binary Crate src/main.rs 有 main()，能运行
Library Crate src/lib.rs 没有 main()，只能被其他代码用

项目结构

  • 唯一库包：src/lib.rs
  • 默认二进制包：src/main.rs，编译后生成的可执行文件与 Package 同名
  • 其余二进制包：src/bin/main1.rs 和 src/bin/main2.rs，它们会分别生成一个文件同名的二进制可执行文件
  • 集成测试文件：tests 目录下
  • 基准性能测试 benchmark 文件：benches 目录下
  • 项目示例：examples 目录下

模块

创建嵌套模块

这段代码用 Rust 的模块系统定义了一个“前厅”模块，里面还有两个子模块，分别是：

  • hosting（接待模块）
  • serving（服务模块）
每个模块的作用
mod front_of_house

这是主模块 “餐厅前厅”。模块（module）是 Rust 的代码组织方式，就像 Java 中的包、Python 中的模块。

mod hosting

接待模块，负责安排顾客进门，比如：

  • 加入等位名单
  • 安排座位
mod serving

服务模块，负责送餐、点餐、结账等，比如：

  • 点餐
  • 上菜
  • 收款

模块树

使用super引用模块

super 代表当前模块的“上一级模块”，也就是父模块; 它就像文件夹里的 ../，表示“返回上一级目录”。

使用self引用模块

self 表示“当前模块自身”就像 self 在方法中表示当前对象一样，在模块路径中，它表示“这个模块的作用域”。

关键字 意义 举例
self 当前模块 self::func()
super 上一级模块 super::helper()
crate 根模块（主模块） crate::config::init()
