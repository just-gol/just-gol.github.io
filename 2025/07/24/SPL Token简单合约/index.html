Token合约

整体结构快速总结

外部依赖包说明

每个包的作用

包名 功能
borsh 用于反序列化客户端传来的指令数据，类似 JSON -> Struct
solana_program 核心库，包括账户管理、消息打印、系统合约、租金等
spl_token Token 标准库，用于创建 Mint、转账、初始化等操作
spl_associated_token_account 用于创建与用户地址相关联的 Token 账户（ATA）

process 方法详解

参数说明

参数 类型 含义
_program_id &Pubkey 当前合约的 program ID（有时用于判断 PDA 属主）
accounts &[AccountInfo] 所有传入的账户列表（顺序必须匹配！）
instruction_data &[u8] 客户端传入的字节数据，序列化的指令

指令解析

  • 使用 borsh::try_from_slice 把客户端传来的字节数组反序列化成枚举 TokenInstruction。
  • 如果数据非法，会直接 return Err(...)，返回 ProgramError::InvalidInstructionData。

匹配枚举并调用相应方法

  • 客户端传的是哪种指令（CreateToken 还是 Mint），就调用相应逻辑。

create_token 方法详解

账户解析（顺序很重要）

顺序 参数名 用途 是否 writable 是否 signer 说明
0 mint_account 要新建的 Mint 账户（即这个 token 的核心信息） ✅ 是 ✅ 是 create_account 创建、initialize_mint 初始化它
1 mint_authority 将被设置为该 token 的 mint 权限（谁能铸币） ❌ 否 ❌ 否（或是） 写入到 Mint 数据中（可以是 payer 本人）
2 payer 出 gas 的人 ❌ 否 ✅ 是 用来付 create_account 的租金和手续费
3 rent_sysvar 系统的租金计算账户 ❌ 否 ❌ 否 用来计算最低 balance
4 system_program 系统合约 ❌ 否 ❌ 否 创建账户时需要它
5 token_program SPL Token 合约 ❌ 否 ❌ 否 初始化 token（initialize_mint）时需要用它

创建 Mint 账户（使用系统程序）

  • 调用 invoke() 发起链上系统程序指令，创建一个新账户。
  • 如果该账户已经存在，会报错。
  • 必须保证 mint_account 是未初始化的 signer 且 writable。

初始化 Mint（铸币定义）

  • 初始化 Mint 结构体（类似于定义这个 Token 的规则）；
  • 设置 authority（谁有权 mint）；
  • 设置 decimal 位数；
  • 设置该账户所属的 owner = spl_token。

mint_token 方法详解

解析账户

顺序 参数名 用途 是否 writable 是否 signer 说明
0 mint_account 之前创建好的 Mint 账户 ❌ 否 ❌ 否 它定义了 Token 是什么，铸币操作会往它写
1 associated_token_account (ATA) 用户钱包对应的 token 持有账户 ✅ 是 ❌ 否 如果不存在就创建它；用于接收 token
2 rent_sysvar 系统的租金账户 ❌ 否 ❌ 否 用来创建 ATA 时计算最小余额
3 payer 出 gas 的人 ✅ 是 ✅ 是 创建 ATA / Mint token 都会用到它
4 system_program 系统合约 ❌ 否 ❌ 否 创建账户时需要
5 token_program SPL Token 合约 ❌ 否 ❌ 否 mint_to 调用时用
6 associated_token_program ATA 专用合约 ❌ 否 ❌ 否 创建 ATA 时必须使用它

判断是否需要创建 ATA（关联 Token 账户）

  • 判断 ATA 是否存在（是否已经创建）
  • 如果没有，就调用 create_associated_token_account 创建 ATA

铸币到 ATA（mint_to）

  • payer 同时是 authority；
  • 执行 mint_to 将指定数量的 Token 铸币到 ATA；

总结

方法 作用 输入账户 关键操作
process 解析指令 指令字节 调用 create_token 或 mint_token
create_token 创建 Mint mint, authority, payer, rent, system_program, token_program create_account + initialize_mint
mint_token 铸币到 ATA mint, ATA, payer, rent, system_program, token_program, associated_token_program create ATA + mint_to

代码整体流程是调用

铸币到用户的 ATA（关联代币账户）

test_fn() 函数

  • 连接本地的 Solana 节点 RPC，发起交易和查询。
  • 读取本地钱包私钥，用它支付手续费，签名交易。
  • 你的链上程序 ID（合约地址），告诉交易调用哪个程序
  • 新生成一个 Mint 账户 Keypair（公钥是这个 Token 的“身份证”）。
  • 这个账户需要链上创建并分配空间。

调用你的自定义函数创建 Token，decimals = 6。

  • 调用铸币函数，给关联账户铸币 1 亿（因为 decimals=6，相当于 100 个 Token）。

create_token(…) 函数细节

参数定义

参数名 说明
rpc_client 用于发送交易到 Solana 节点
program_id 你的链上程序 ID
payer 付款账户，付手续费和创建账户费用
mint_account 新创建的 Mint 账户 Keypair
mint_authority 这个 Token 的铸币权限公钥
decimals Token 精度，小数点位数

指令序列化

  • 你定义的 TokenInstruction::CreateToken 结构体通过 Borsh 序列化成字节数组，链上程序根据它解析调用逻辑。

账户元数据构造

  • 这就是链上程序接收的账户列表，顺序和角色必须和链上程序 Processor::create_token 里 next_account_info 顺序对应。

构造交易指令

  • 构造一个调用你程序的 Instruction。

创建并签名交易

  • 获取最近区块哈希防止重放攻击；
  • 创建交易，签名者是 payer 和 mint_account（mint_account 是新账户，需要签名证明是本人创建）；
  • payer 是手续费付出者。

发送并确认交易

  • 把交易发送到链上节点，等待交易确认；
  • 输出交易 ID。

mint(…) 铸币函数细节

计算 ATA 地址

  • 根据 payer 和 mint 计算 ATA（关联 Token 账户）地址；
  • ATA 是用户存放某种 Token 的账户。

指令序列化

  • 铸币数量序列化。

构造账户元数据

  • 传入你链上程序中 mint_token 处理函数对应账户。

构造并签名交易

  • 发送交易，签名者是 payer 和 mint_account。

发送交易，确认成功
