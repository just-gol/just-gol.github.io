API

Path::new(FILE_PATH)

- 创建一个 Path 对象，其中 FILE_PATH 是一个字符串或字符串切片（&str 或 String），表示文件系统路径。

.exists() - 这是 Path 结构体的方法，它会：

- 返回一个布尔值`（bool）`
- 如果路径指向的文件或目录确实存在，返回`true`
- 如果路径不存在或程序没有权限访问该路径，返回`false`

fs::read_to_string

1.用于将整个文件内容读取为一个 String。它属于标准库的 std::fs 模块，适用于读取文本文件（如 .txt、.rs、.json 等）

serde_json::from_str

- JSON 字符串反序列化为 Rust 数据结构

serde_json::to_string_pretty

- Rust 数据结构序列化为格式化的 JSON 字符串

作用：将 Rust 数据结构转换为格式化的（带缩进和换行的）JSON 字符串，适合人类阅读。

env::args()

  • 返回一个实现了 Iterator 的 Args 迭代器，包含程序启动时传入的命令行参数。
    • 第一个参数（索引 0）通常是程序本身的路径（如 target/debug/my_program）。
    • 后续参数是用户输入的参数（如 arg1 arg2）。
  • .collect()
    将迭代器收集到指定的集合类型（这里是 Vec<String>）。
  • .as_str()
    是 String 类型的方法，返回该字符串的 不可变引用（&str）。

添加的依赖

  • **<font style="color:rgb(64, 64, 64);">serde</font>**
    • 功能：提供序列化/反序列化的框架，支持多种数据格式（JSON、YAML 等）。
    • features = ["derive"]：启用派生宏，允许通过 #[derive(Serialize, Deserialize)] 自动为结构体生成序列化代码。
  • serde_json
    • 功能：专门处理 JSON 格式的序列化/反序列化实现，依赖 serde 的接口。
库 作用 典型用途
serde 定义序列化接口和派生宏 结构体/枚举的序列化能力声明
serde_json JSON 格式的具体实现 实际解析/生成 JSON 数据
